Introduction

Welcome to the FinchXP API reference documentation.

The FinchXP API provides secure access to data intelligence around Australian transactions - ingesting millions of transactions every month to provide you with industry-leading identification, categorisation, and additional information.

SingleTransactionParseRequest: object

transactionId: string

The transaction number is a unique transaction number as defined by the API Consumer. Each transaction belongs to only one account.

transactionDescription: string

The original description of the transaction as provided by the institution.

providerId: string

Internal provider ID to parse

transactionDate: string (date-time)

The original date of the transaction as provided by the institution.

transactionAmount: number

The signed original amount of the transaction as provided by the institution.

transactionCurrency: string

The currency of the transaction as provided by the institution, default AUD.

transactionStatus: string POSTED, PENDING

The status of the transaction, can be POSTED or PENDING.

transactionSource: string

The source of the transactions, can be aggregator name (i.e. Yodlee, Proviso) or Bank

userId: string

Internal User ID to the API Consumer's platform

accountNumber: string

The account number. This will typically be a masked or partial account number.

accountType: string

An account type represents a financial account held by a financial institution, e.g., a user's checking, savings, or investment account.

accountSubType: string

An account subtype represents the specific type that belongs to an account type, usually loan subtypes.

refreshDate: string (date-time)

The date the account was last refreshed. If the value is empty, null will be returned.

balanceDate: string (date-time)

The last time the balance was refreshed. If the value is empty, null will be returned.

accountBalance: number

The balance in an account, pre-holds or pending transactions. Max length is 14,2. If the value is empty, null will be returned.

availableBalance: number

The available balance in an account, pre-holds or pending transactions. Max length is 14,2. If the value is empty, null will be returned.

transactionType: string CREDIT, DEBIT

A transaction type represents a money movement in or out of an account e.g. money in, income, or credit; money out, expense, or debit.

Example
{
  "transactionId": "61",
  "transactionDescription": "Discount Drug Beachme    Beachmere    AU",
  "providerId": "CBAI",
  "transactionDate": "2020-11-21",
  "transactionAmount": "150.00",
  "transactionCurrency": "AUD",
  "transactionStatus": "POSTED",
  "transactionSource": "Bank",
  "userId": "U0001",
  "accountNumber": "U0001",
  "accountType": "Saving",
  "accountSubType": "",
  "refreshDate": "2020-11-21",
  "balanceDate": "2020-11-21",
  "accountBalance": "number",
  "availableBalance": "number",
  "transactionType": "DEBIT"
}

JobCreateRequestModel: object

contentType: string text/csv, text/tab-separated-values, application/x-lzop, application/json, application/gzip, application/x-snappy-framed
format: string csv, tsv, json, orc, parquet
processingType: string standard, expedited

Definding how job should be processed. standard will process it at normal rates, expedited will do so at faster rates at higher cost

enrichmentLevel: string BASIC, CATEGORISATION, PRO

Level of enrichment to be returned. This will control the fields to be returned and the pricing per API call will also change accordingly

expiresIn: number

How long the signed URL to upload job should be valid for. If blank, this will use the system default

requireMultipart: boolean

If True, multipart upload will be initiated. Mandatory for 5GB file upload or above, recommended for 100MB+ uploads. The SignedURL returned will be the link to upload the first part. Call to getPartUrl is required for the next part then completeUpload is required after all parts are uploaded.

notifiedWebhook: string

Optional WebHook to call when job is completed or failed

notifiedEmail: string

Optional Email to receive notification when the job is completed or failed

Example
{
  "contentType": "string",
  "format": "string",
  "processingType": "string",
  "enrichmentLevel": "BASIC",
  "expiresIn": 86400,
  "requireMultipart": false,
  "notifiedWebhook": "",
  "notifiedEmail": ""
}
